# World Cup 2026 feed sample, a hand-built test fixture (NOT the live feed).
# It mirrors the openfootball cup.txt format and deliberately packs the edge cases that
# have bitten the parsers, so both test_app.js and test_pipeline.py can run a feed end to end:
#   - a penalty written as the shorthand (p)   -> must credit the scorer, not invent a player "p"
#   - an own goal written (og)                 -> must NOT credit the scorer
#   - a scorer block that spans several lines   -> must be gathered whole
#   - a played game with NO half-time score     -> must still parse, not be dropped
#   - an unplayed " v " fixture                 -> must be skipped, not invented
#   - an official FIFA name (Côte d'Ivoire)     -> must map to the engine name (Ivory Coast)
#   - a knockout tie (a 1-1 draw)               -> must split off from the group rows, winner unknown
